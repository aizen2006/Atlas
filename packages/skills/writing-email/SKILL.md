# Email Writing Skill

## Purpose

This skill specializes in producing high-quality emails that are clear, professional, concise, and appropriate for the recipient and context.

Always optimize for clarity, readability, and actionability.

---

# When to use this Skill

Use this skill whenever the user asks to:

- Write an email
- Draft an email
- Reply to an email
- Improve an existing email
- Rewrite an email
- Make an email shorter or longer
- Make an email more professional
- Make an email friendlier
- Write follow-up emails
- Write sales emails
- Write cold outreach
- Write customer support emails
- Write recruitment emails
- Write business proposals
- Write project updates
- Write status updates
- Write thank-you emails
- Write apology emails
- Write interview emails
- Write partnership emails

---

# Core Principles

Every email should:

- Have a clear purpose.
- Respect the recipient's time.
- Avoid unnecessary filler.
- Be easy to skim.
- End with a clear next action whenever appropriate.
- Sound natural rather than AI-generated.
- Match the requested tone.

Never use excessive corporate jargon unless explicitly requested.

---

# Workflow

Follow these steps.

## Step 1 — Determine intent

Identify the email category.

Possible categories:

- Cold Outreach
- Sales
- Business
- Internal Team
- Follow-up
- Customer Support
- Job Application
- Networking
- Interview
- Partnership
- Personal
- Thank You
- Apology
- Announcement
- Other

---

## Step 2 — Identify recipient

Determine who the email is written to.

Examples:

- Client
- Founder
- CEO
- Hiring Manager
- Recruiter
- Investor
- Team Member
- Friend
- Vendor
- Customer

Adapt language accordingly.

---

## Step 3 — Determine tone

Use the user's requested tone.

Possible tones:

- Professional
- Friendly
- Formal
- Casual
- Confident
- Persuasive
- Empathetic
- Direct
- Executive
- Concise

If no tone is specified, default to:

Professional, warm, concise.

---

## Step 4 — Organize the email

Unless the user requests another structure, use:

Greeting

Opening

Purpose

Supporting details

Call to action

Closing

Signature (only if requested)

---

## Step 5 — Improve readability

Prefer:

Short paragraphs.

Bullet points when listing information.

Simple language.

Active voice.

Strong verbs.

Avoid long walls of text.

---

# Subject Line Guidelines

Create a concise and relevant subject.

Examples:

Project Update

Following Up

Quick Question

Proposal for Collaboration

Application for Full Stack Developer

Meeting Follow-up

Thank You

Avoid clickbait.

---

# Reply Guidelines

When replying:

- Answer every question.
- Preserve important context.
- Address concerns directly.
- Keep the conversation moving.
- Avoid repeating unnecessary information.

---

# Cold Outreach Guidelines

Cold emails should:

- Personalize the opening.
- Explain why you're reaching out.
- Clearly communicate value.
- Keep the email under 200 words unless requested otherwise.
- Include one clear CTA.

Avoid sounding spammy.

---

# Follow-up Guidelines

Follow-ups should:

- Reference the previous interaction.
- Add new value whenever possible.
- Be polite.
- Never sound demanding.
- Include one clear next step.

---

# Professional Writing Rules

Always:

Use proper grammar.

Use proper punctuation.

Use complete sentences.

Avoid emojis unless requested.

Avoid slang unless requested.

Avoid unnecessary capitalization.

Avoid multiple exclamation marks.

---

# Formatting Rules

Prefer this format:

Subject

Greeting

Body

Closing

Signature (optional)

Separate paragraphs with blank lines.

Use bullet points for multiple items.

---

# If Information Is Missing

If essential information is missing, infer only when it is safe.

Otherwise ask concise clarification questions such as:

- Who is the recipient?
- What is the purpose?
- What tone would you like?
- Is there a deadline?
- Would you like a subject line?

Do not invent factual details.

---

# Quality Checklist

Before returning the email, verify:

- Purpose is clear.
- Tone matches the request.
- Grammar is correct.
- Subject is relevant.
- Call-to-action is clear.
- No redundant sentences.
- Easy to skim.
- Sounds natural.

---

# Output Format

Return emails using the following structure.

Subject:
<subject>

Hi <recipient>,

<body>

Best,
<name if provided>
